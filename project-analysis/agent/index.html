Agent 核心循环

整个项目的心脏。一条用户消息进来,如何驱动「调用模型 → 执行工具 → 再调用模型」的多轮循环,直到产出最终答复。本模块拆成 5 篇逐层深入,建议按顺序读。

这块代码在哪

核心循环的代码并不在一个文件里,而是「一个薄入口 + 一个真正的实现 + 若干被抽出的协作模块」:

文件角色
run_agent.pyAIAgent.run_conversation() / chat() —— 对外的薄入口,转发到真正的实现
agent/conversation_loop.py真正的主循环:准备 → while 循环 → 收尾,本模块的主角
agent/turn_context.py把「每轮一次」的准备工作(build_turn_context)从循环里抽出来
agent/tool_executor.py工具调用的并发 / 串行执行
agent/iteration_budget.py迭代预算,循环的安全阀
agent/turn_finalizer.py收尾:持久化、后置钩子、后台复盘
阅读策略:run_agent.py 有 ~12k 行,别从头读。本模块只追 run_conversation() 这一条主线;它在 run_agent.py 里只是一个转发函数,真身在 agent/conversation_loop.py。

一句话全貌

run_conversation() = 一次准备(建好 system prompt、压缩、落库)+ 一个 while 循环(反复「问模型 → 跑工具 → 回灌结果」)+ 一次收尾(落库、钩子、后台复盘)。循环用迭代预算和 max_iterations 当安全阀,用中断标志响应 /stop。

本模块文档

1
入口与准备阶段
从 chat() 到 run_conversation();build_turn_context 一次性做完 system prompt 恢复、压缩、落库、记忆预取。
run_agent.py · conversation_loop.py · turn_context.py
2
主循环机制
while 条件、迭代预算与 grace 调用、中断检查、step_callback —— 循环如何转、如何安全退出。
conversation_loop.py · iteration_budget.py
3
调用模型
发请求前对消息做的一切:缓存断点、清洗、KV-cache 友好的规范化、失败重试与故障转移。
conversation_loop.py · prompt_caching.py · chat_completion_helpers.py
4
工具执行
拿到 tool_calls 后:并发 vs 串行、handle_function_call 分发、结果回灌、审批与守卫。
tool_executor.py · model_tools.py · tools/registry.py
5
收尾阶段
循环结束后:持久化轮次、post 钩子、后台记忆/技能复盘、返回结果字典。
turn_finalizer.py · conversation_loop.py
